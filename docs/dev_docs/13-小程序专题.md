# 小程序专题

uni-app 在支持 Vue 组件之外，也兼容了各家小程序的自定义组件规范。

## 平台差异说明

| 平台 | 支持情况 | 组件存放目录 |
|------|---------|-------------|
| H5 | Vue2 支持微信组件，Vue3 不支持 | wxcomponents |
| App（不含 nvue） | Vue2 支持微信组件，Vue3 不支持 | wxcomponents |
| 微信小程序 | 支持 | wxcomponents |
| 支付宝小程序 | 支持 | mycomponents |
| 百度小程序 | 支持 | swancomponents |
| 抖音、飞书小程序 | 支持 | ttcomponents |
| QQ 小程序 | 支持 | wxcomponents |
| 快手小程序 | 支持 | kscomponents |
| 京东小程序 | 支持 | jdcomponents |
| 鸿蒙元服务 | 支持 | hascomponents |
| 小红书小程序 | 支持 | xhscomponents |

## 使用方式

在 `pages.json` 对应页面的 style → usingComponents 中引入组件：

```json
{
    "pages": [{
        "path": "index/index",
        "style": {
            "usingComponents": {
                "custom": "/wxcomponents/custom/index"
            }
        }
    }]
}
```

在页面中引用：

```html
<template>
  <view>
    <custom name="uni-app"></custom>
  </view>
</template>
```

## 注意事项

- 小程序组件性能不如 Vue 组件——小程序需手动 setData，而 Vue 组件会自动 diff 更新差量数据
- 数据与事件绑定的差异，应按 Vue 的方式绑定：
  - 属性绑定：`attr=""` → `:attr="a"`
  - 事件绑定：`bind:click="fn"` → `@click="fn"`
  - 阻止冒泡：`catch:tap="xx"` → `@tap.native.stop="xx"`
  - `wx:if` → `v-if`
  - `wx:for="" wx:key=""` → `v-for="(item,index) in list"`

# WXS

WXS 是一套运行在视图层的脚本语言。其特点是运行在视图层，可减少逻辑层与渲染层之间的通信折损。

## 平台差异说明

| App | H5 | 微信小程序 | 支付宝小程序 | 百度小程序 | 抖音/飞书小程序 | QQ 小程序 |
|-----|----|-----------|-------------|-----------|---------------|----------|
| √（不支持 nvue） | √ | √ | SJS | Filter | SJS | √ |

> App 端 nvue 应使用 bindingx 而非 wxs；App 端 vue 推荐使用更强大的 renderjs。

## WXS 示例

```html
<template>
  <view>
    <view class="area">
      <view @touchstart="test.touchstart" @touchmove="test.touchmove" class="movable">
        {{test.msg}}
      </view>
    </view>
  </view>
</template>

<script module="test" lang="wxs">
var startX = 0
var startY = 0
var lastLeft = 50; var lastTop = 50

function touchstart(event, ins) {
  var touch = event.touches[0] || event.changedTouches[0]
  startX = touch.pageX
  startY = touch.pageY
}

function touchmove(event, ins) {
  var touch = event.touches[0] || event.changedTouches[0]
  var pageX = touch.pageX
  var pageY = touch.pageY
  var left = pageX - startX + lastLeft
  var top = pageY - startY + lastTop
  startX = pageX
  startY = pageY
  lastLeft = left
  lastTop = top
  ins.selectComponent('.movable').setStyle({
    left: left + 'px',
    top: top + 'px'
  })
  return false
}

module.exports = {
  msg: 'Hello',
  touchstart: touchstart,
  touchmove: touchmove
}
</script>
```

## 引入方式

```html
<!-- 内联 -->
<script module="test" lang="wxs">
  //...code
</script>

<!-- 外部引入 -->
<script module="utils" lang="wxs" src="./utils.wxs"></script>
<script module="utils" lang="filter" src="./utils.filter.js"></script>
<script module="utils" lang="sjs" src="./utils.sjs"></script>
```

## 重要注意事项

- module 指定的模块名不可与 data、methods、computed 内的属性重名
- Vue3 项目不支持 `<script setup>` 用法
- 各小程序规范仍在完善中，请仔细阅读对应平台文档
- 暂不支持在 wxs、sjs、filter.js 中调用其他同类型文件
- nvue 页面暂不支持 wxs、sjs、filter.js
- App 和 H5 端提供 wxs 的升级版 renderjs，功能更强大
