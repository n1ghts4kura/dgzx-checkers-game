# 条件编译

## 为什么选择条件编译

uni-app 已将常用组件、API 封装到框架中，按规范开发即可保证多平台兼容。但每个平台有其自身特性，存在一些无法跨平台的情况。uni-app 提供了条件编译手段，让开发者在一个工程里完成平台个性化实现。

## 使用方法

以 `#ifdef` 或 `#ifndef` 加 `%PLATFORM%` 开头，以 `#endif` 结尾。

- **`#ifdef`**：if defined，仅在某平台存在
- **`#ifndef`**：if not defined，除某平台外均存在
- **`%PLATFORM%`**：平台名称

| 写法 | 说明 |
|---|---|
| `#ifdef APP-PLUS` ... `#endif` | 仅出现在 App 平台 |
| `#ifndef H5` ... `#endif` | 除 H5 平台外 |
| `#ifdef H5 \|\| MP-WEIXIN` ... `#endif` | H5 或微信小程序 |

## %PLATFORM% 可取值

| 值 | 生效条件 |
|---|---|
| `VUE3` | uni-app js 引擎版区分 vue2 和 3 |
| `VUE2` | uni-app js 引擎版区分 vue2 和 3 |
| `APP` | App 平台 |
| `APP-PLUS` | uni-app js 引擎版编译为 App 时 |
| `APP-PLUS-NVUE` 或 `APP-NVUE` | App nvue 页面 |
| `APP-ANDROID` | App Android 平台 |
| `APP-IOS` | App iOS 平台 |
| `APP-HARMONY` | App HarmonyOS Next 平台 |
| `H5` | H5（推荐使用 `WEB`） |
| `WEB` | web（同 `H5`） |
| `MP-WEIXIN` | 微信小程序 |
| `MP-ALIPAY` | 支付宝小程序 |
| `MP-BAIDU` | 百度小程序 |
| `MP-TOUTIAO` | 抖音小程序 |
| `MP-LARK` | 飞书小程序 |
| `MP-QQ` | QQ 小程序 |
| `MP-KUAISHOU` | 快手小程序 |
| `MP-JD` | 京东小程序 |
| `MP-360` | 360 小程序 |
| `MP-HARMONY` | 鸿蒙元服务 |
| `MP-XHS` | 小红书小程序 |
| `MP` | 所有小程序 |
| `QUICKAPP-WEBVIEW` | 快应用通用 |

## 支持的文件类型

- `.vue` / `.nvue` / `.uvue`
- `.js` / `.uts`
- `.css`
- `pages.json`
- 各预编译语言文件，如 `.scss`、`.less`、`.stylus`、`.ts`、`.pug`

## 注意事项

- 条件编译利用注释实现，不同语法中注释写法不同：js/uts 用 `//`，css 用 `/* */`，vue/nvue/uvue 模板用 `<!-- -->`
- 使用条件编译需保证编译前后的文件语法正确性

## 各区域的示例

### API 的条件编译

```js
// #ifdef APP
console.log('仅App平台输出')
// #endif

// #ifndef H5
console.log('H5平台不输出')
// #endif

// #ifdef APP || H5
console.log('App和H5平台均输出')
// #endif
```

### 组件的条件编译

```html
<!-- #ifdef MP-WEIXIN -->
<official-account></official-account>
<!-- #endif -->
```

### 样式的条件编译

```css
/* #ifdef APP */
.app-only { color: red; }
/* #endif */
```

> 无论使用 css 还是 sass/scss/less/stylus，**必须使用 `/* */` 注释写法**。

### pages.json 的条件编译

```json
{
  "pages": [
    // #ifdef APP
    {
      "path": "pages/app-only/index",
      "style": {}
    }
    // #endif
  ]
}
```

### static 目录的条件编译

在 `static` 目录下新建不同平台的专有目录（目录名称为**小写**）：

| 目录名称 | 说明 |
|---|---|
| `app-plus` / `app` | App |
| `h5` / `web` | Web |
| `mp-weixin` | 微信小程序 |
| `mp-alipay` | 支付宝小程序 |
| `mp-baidu` | 百度小程序 |
| `mp-qq` | QQ 小程序 |
| `mp-toutiao` | 抖音小程序 |
| `mp-lark` | 飞书小程序 |
| `mp-kuaishou` | 快手小程序 |
| `mp-harmony` | 鸿蒙元服务 |
| `mp-xhs` | 小红书小程序 |
| `mp-jd` | 京东小程序 |

专有目录下的静态资源仅在特定平台编译进去。

## 整体目录条件编译

在 uni-app 项目根目录创建 `platforms` 目录，然后在其中创建 `app-plus`、`mp-weixin` 等子目录，存放不同平台的文件。`platforms` 目录下只支持放置页面文件。

## uni-app x 项目的条件编译

```js
// #ifdef UNI-APP-X
// uni-app x 项目的代码
// #endif
// #ifndef UNI-APP-X
// uni-app 项目的代码
// #endif
```

## HBuilderX 支持

- 输入 `ifdef` 可快速生成条件编译代码片段
- 条件编译注释部分提供语法高亮
- `Ctrl+Alt+/` 可生成正确注释
- 点击 `ifdef` 或 `endif` 可快速选中条件编译部分
